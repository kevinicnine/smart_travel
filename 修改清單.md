# 專題報告修改清單

此檔為依據專案新功能與架構變更，針對原始《專題成果報告》需要檢視與更新的項目清單，含優先順序與建議改寫方向。

## 總覽
- 目標：讓報告反映目前系統架構、功能、實作細節與評估結果的最新狀態。
- 優先順序：A = 必要（報告中誤導或已過時）、B = 建議（提升可讀性或完整性）、C = 可選（細節或補充）。

## 優先修正項目（A）
- **摘要 / 目的**：更新專案目標與新增功能摘要，強調目前可用功能與差異化特色。
- **整體架構圖與文字說明**：替換或補充系統架構圖（新增 ETL、API、模型與前端路由等變動），更新各元件責任說明與通訊流程。
- **功能清單與範例流程**：把原本功能列表與使用流程和新加入的功能（例如：關鍵字分類、推薦流程、前端使用者流程）同步。
- **部署與環境說明**：補充 `.env`、Docker、服務啟動步驟與必要外部資源（資料庫、向量索引、模型檔）的變更。

## 建議修正項目（B）
- **技術細節**：更新使用到的套件/版本、模型架構、特徵工程流程、資料清理與儲存位置。
- **API 介面文件**：列出已新增或變動的重要 API 路由、輸入輸出格式與範例（可附在附錄）。
- **資料管線（ETL）說明**：補充原始資料來源、預處理步驟、排程與錯誤處理策略。
- **測試與評估**：新增最近的評估結果、測試覆蓋率、延遲/吞吐量基準（或標註仍需補測項）。

## 可選補充項（C）
- **使用者案例 / UX 改良紀錄**：新增使用流程示意與 UI 改動截圖。
- **安全性與權限設計**：列出認證/授權機制（如有）與敏感資料處理摘要。
- **未來工作與限制**：補充因新增功能而引入的技術債或限制，以及可能的改進方向。

## 逐章節對應修正建議（可直接複製貼上）

- **封面 / 摘要**（A）：把摘要改為 3–5 句，說明「專案目的」、「主要功能」、「目前進度（如：已完成後端 API、上線模型、前端頁面）」。

- **目錄**：確認頁碼與新增章節（如：API 附錄、部署步驟）對應。

- **第一章：緒論**（B）
  - 更新動機與問題指向，說明新增功能如何解決原需求。

- **系統分析 / 架構**（A）
  - 換圖或重畫：加入後端 `smart-travel-api`、資料 ETL、模型訓練/向量索引、前端 `smart-travel-frontend` 與服務間互動箭頭。
  - 補充資料流程（來源 → 預處理 → 特徵 → 儲存/索引 → 查詢）。

- **功能實作 / 模組說明**（A/B）
  - 每個模組列出：位置（檔案或資料夾，如 `smart-travel-api/app/`、`smart-travel/app/etl/`）、主要責任、輸入/輸出。
  - API 範例：列出 3 個核心路由（例如：關鍵字查詢、行程推薦、資料匯入）。

- **資料與模型**（B）
  - 更新資料集來源、標注方法、分割策略、模型選擇與訓練參數摘要。

- **實驗與評估**（B）
  - 新增最近結果（精確度、召回、回應延遲）。若尚未測量，標註需補測指標。

- **部署與維運**（A）
  - 列出部署步驟（Docker、env 變數、DB 初始化、遷移指令）、監控與日誌位置（backend.log 範例）。

- **結論與未來工作**（C）
  - 更新完成項目、未完成項與短中期計畫。
